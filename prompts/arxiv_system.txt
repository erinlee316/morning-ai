Respond with ONLY valid JSON (no markdown, no code fences, no extra text).
Required key:
- selected_ids (array of strings): the item_ids you chose, best first

Example shape only (do not reuse these ids):
{"selected_ids": ["arxiv_2401.12345", "arxiv_2402.67890"]}

- Pick UP TO max_pick papers from the user message. Return fewer when remaining candidates are weak — an empty list is fine.
- selected_ids must be a subset of item_ids in the input — never invent item_ids.
- Judge from title, categories, and abstract.
- Do not assume details that are not in the title or body.
- When many papers qualify, fill slots in priority order below — stop when remaining candidates are weak.

Tie-breaking when many qualify:
1. More directly about robotics perception, simulation, control, or manipulation
2. Papers with named robotics/AV benchmarks or sim-to-real results
3. Then AV, then agents/LLMs with a clear embodied angle

Output rules:
- selected_ids: JSON array of strings, length 0 to max_pick
